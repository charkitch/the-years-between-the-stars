# Time in Transit

A browser-based space exploration and trading game inspired by classics like Elite and the works of Vernor Vinge and Alastair Reynolds. Fly your ship across a procedurally generated galaxy, trade goods between stations, navigate faction politics, and watch civilizations rise and fall as relativistic time passes with each hyperspace jump.

Built with React, TypeScript, Three.js, and Vite — with a Rust/WASM game engine for procedural generation, galaxy simulation.

## Features

- **Free-flight spaceship controls** — thrust, pitch, yaw, roll, with shield, heat, and fuel management
- **Trading economy** — buy and sell goods (food, textiles, luxuries, narcotics, and more) across 30 star systems
- **Hyperspace travel** — jump between systems with relativistic time dilation; years pass with each jump
- **Faction politics** — 6 factions vie for control of systems, with contested territories and fleet battles
- **Procedural generation** — seeded galaxy with multiple star types, orbiting planets, and stations
- **Galaxy simulation** — the galaxy evolves while you travel; factions grow, economies shift, and your choices ripple forward
- **Rust/WASM engine** — game logic compiled to WebAssembly for real spoiler protection and a living galaxy
- **Docking and stations** — dock at stations to trade, refuel, and interact with NPCs

## Getting Started

### Prerequisites

- Node.js 18+ (or newer LTS)
- npm
- Rust toolchain with `wasm32-unknown-unknown` target (`rustup target add wasm32-unknown-unknown`)
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/)

### Run the Game

```bash
npm install
npm run wasm:build    # compile the Rust engine to WASM
npm run dev
```

Then open the local URL printed by Vite (typically `http://localhost:5173`).

### Build for Production

```bash
npm run build         # builds WASM + TypeScript + Vite
npm run preview
```

## Architecture

```
src/          — TypeScript frontend (React UI, Three.js rendering, flight physics)
engine/       — Rust/WASM game engine (generation, simulation, events, trading)
engine/pkg/   — WASM build output (auto-generated by wasm-pack)
```

The Rust engine handles all procedural generation, galaxy simulation, event selection, and market pricing. The TypeScript frontend handles rendering, flight physics (60fps), and UI. They communicate through a JSON boundary — see [`engine/README.md`](./engine/README.md) for details.
